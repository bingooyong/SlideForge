## 1. 监控目标

- 当前无内置监控系统；建议至少对后端健康状态做探测，便于运维与负载均衡。

## 2. 健康检查

- **端点**：`GET /health`（若已实现）；返回 200 表示服务存活。
- **用途**：负载均衡探活、容器编排 liveness/readiness。

## 3. 告警与 Runbook

- 告警规则与阈值待补充（如：健康检查连续失败、磁盘空间、Gemini 配额）。
- 故障处置见 [runbooks/](runbooks/)（待补充单则 Runbook）。

## 4. 日志

- 后端使用 structlog；`LOG_LEVEL` 控制级别；日志输出位置见 [configuration.md](configuration.md) 与 backend 配置。
- 生产建议配置日志轮转与集中收集（按需）。
